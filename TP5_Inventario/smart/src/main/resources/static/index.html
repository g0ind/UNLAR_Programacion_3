Justificación Algorítmica

Las operaciones de inserción y búsqueda directa por ID son O(1) porque operan directamente sobre claves en una tabla hash segura para hilos (ConcurrentHashMap). Por otro lado, los filtros y listados iteran todos los elementos aplicando predicados lógicos, por lo que escalan de manera lineal O(n). El ordenamiento en Java utiliza TimSort, lo cual crece a un orden de O(n log n).

💡 Tip para el reporte: La primera prueba puede reflejar tiempos mayores de ejecución debido al overhead inicial del compilador JIT de la JVM. Se sugiere realizar múltiples ejecuciones para promediar las estadísticas en condiciones estables.
